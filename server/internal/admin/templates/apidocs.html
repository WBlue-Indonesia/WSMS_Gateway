Authentication

Client API key

Every /v1 client endpoint requires a bearer token.

Authorization: Bearer wsms_<prefix>.<secret>

Create and revoke keys under Clients & Keys. Only a hash is stored — the secret is shown once, at creation. A key carries scopes (e.g. messages:write).

Optional request signing (HMAC)

If a key has signing enabled, add these headers to every request:

X-WSMS-Timestamp: 1752537600
X-WSMS-Signature: sha256=HMAC_SHA256(secret, timestamp + "." + body)

The timestamp must be within ±5 minutes of server time.

Sender phones authenticate on the WebSocket with Authorization: Bearer dev_<device_id>.<device_secret>. Rate limit: {{.RateLimit}} — over-limit bursts get 429 with Retry-After.
